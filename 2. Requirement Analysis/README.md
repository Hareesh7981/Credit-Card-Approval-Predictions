# Requirement Analysis

## Overview

The Requirement Analysis phase focuses on identifying and documenting the functional and non-functional requirements needed to develop the Credit Card Approval Prediction System. This phase ensures that the project objectives, system functionality, hardware, software, and user requirements are clearly understood before development begins.

## Purpose

The main objectives of this phase are:

- Understand the project requirements.
- Identify the technologies required for implementation.
- Define the functional and non-functional requirements.
- Analyze system feasibility.
- Prepare the project for the design and development phases.

## Contents

This folder contains the following documents:

- **Functional Requirements.md** – Describes the features and functionalities of the system.
- **Non-Functional Requirements.md** – Defines performance, security, usability, and reliability requirements.
- **Software Requirements.md** – Lists the software tools, libraries, and frameworks used.
- **Hardware Requirements.md** – Specifies the hardware needed to develop and run the project.
- **Requirements Specification.md** – Summarizes all project requirements.

## Functional Requirements

- Upload and process applicant data.
- Perform data preprocessing and feature engineering.
- Train and evaluate machine learning models.
- Predict credit card approval status.
- Display prediction results through a Flask web application.

## Non-Functional Requirements

- Fast prediction response.
- High prediction accuracy.
- Easy-to-use interface.
- Reliable system performance.
- Secure handling of user data.

## Expected Outcome

At the end of this phase, all project requirements are clearly documented, providing a strong foundation for designing, developing, testing, and deploying the Credit Card Approval Prediction system.
